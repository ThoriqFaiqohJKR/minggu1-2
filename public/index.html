FITUR

Konfigurasi kustom dan preset

Deploy aplikasi dalam hitungan menit dengan sampel bawaan yang disebut Solusi Praktis. Buat situs dinamis, VM dengan load balancing, aplikasi web tiga tingkat, atau aplikasi web e-commerce.

Pilih dari jenis, ukuran, dan konfigurasi mesin yang telah ditetapkan untuk workload apa pun, dari aplikasi perusahaan besar hingga workload modern (seperti container) atau project AI/ML yang memerlukan GPU dan TPU.

Untuk fleksibilitas lebih besar, buat jenis mesin kustom antara 1 hingga 96 vCPU dengan memori hingga 8,0 GB per core. Dan manfaatkan salah satu dari banyak opsi block storage, mulai dari Persistent Disk yang fleksibel hingga SSD Lokal berperforma tinggi dan berlatensi rendah.

Keandalan terdepan di industri

Compute Engine menawarkan SLA ketersediaan komputasi instance tunggal terbaik dari penyedia cloud mana pun: ketersediaan 99,95% untuk VM yang dioptimalkan untuk memori dan 99,9% untuk semua kelompok VM lainnya.

Apakah periode nonaktif membuat Anda terjaga di malam hari? Pertahankan kontinuitas workload selama peristiwa yang direncanakan dan tidak direncanakan dengan migrasi langsung. Saat VM berhenti berfungsi, Compute Engine akan melakukan migrasi langsung ke host lain di zona yang sama.

Otomatisasi dan rekomendasi untuk efisiensi resource

Menambahkan VM secara otomatis untuk menangani beban puncak dan mengganti instance berperforma buruk dengan grup instance terkelola.

Sesuaikan resource Anda secara manual menggunakan data historis dengan rekomendasi peny esuaian ukuran, atau jaminan kapasitas untuk lonjakan permintaan yang direncanakan dengan pemesanan untuk masa mendatang.

Semua instance komputasi terbaru kami (C3, A3, H3) berjalan di Titanium, sebuah sistem yang terdiri dari mikrokontroler yang dibuat khusus dan pengurangan penyebaran skala bertingkat untuk meningkatkan performa infrastruktur, pengelolaan siklus proses, dan keamanan Anda.

Harga dan diskon yang transparan

Baca panduan harga mendetail untuk mengetahui jenis atau konfigurasi VM apa pun, atau gunakan kalkulator harga kami untuk mendapatkan estimasi yang dipersonalisasi.

Untuk menghemat tugas batch dan workload yang fault-tolerant, gunakan Spot VM untuk mengurangi tagihan Anda dari 60-91%.

Dapatkan diskon otomatis untuk penggunaan berkelanjutan, atau diskon hingga 70% jika Anda mendaftar diskon abonemen.

Kontrol dan konfigurasi keamanan

Enkripsi data aktif dan saat sedang diproses dengan Confidential VMs.

Bertahan dari rootkit dan bootkit dengan Shielded VM.

Memenuhi standar kepatuhan yang ketat untuk residensi, kedaulatan data, akses, dan enkripsi dengan Assured Workloads.

Workload Manager

Kini tersedia untuk workload SAP, Workload Manager mengevaluasi workload aplikasi Anda dengan mendeteksi penyimpangan dari standar terdokumentasi dan praktik terbaik untuk mencegah masalah secara proaktif, terus menganalisis workload, dan menyederhanakan pemecahan masalah sistem.

VM Manager

VM Manager adalah serangkaian alat yang dapat digunakan untuk mengelola sistem operasi untuk fleet virtual machine (VM) besar yang menjalankan Windows dan Linux di Compute Engine.

Node tenant tunggal

Sole-tenant node adalah server Compute Engine fisik yang disediakan secara eksklusif untuk penggunaan Anda. Sole-tenant node menyederhanakan deployment untuk aplikasi bring your own license (BYOL). Sole-tenant node memberi Anda akses ke jenis mesin dan opsi konfigurasi VM yang sama seperti instance komputasi reguler.

Akselerator TPU

Cloud TPU dapat ditambahkan untuk mempercepat machine learning dan aplikasi kecerdasan buatan. Cloud TPU dapat dipesan, digunakan sesuai permintaan, atau tersedia sebagai preemptible VM.

Dukungan Linux dan Windows

Jalankan OS pilihan Anda, termasuk Debian, CentOS Stream, Fedora CoreOS, SUSE, Ubuntu, Red Hat Enterprise Linux, FreeBSD, atau Windows Server 2008 R2, 2012 R2, dan 2016. Anda juga bisa menggunakan image yang ada di komunitas Google Cloud atau milik Anda sendiri.

Dukungan container

Jalankan, kelola, dan orkestrasikan container Docker di VM Compute Engine dengan Google Kubernetes Engine.

Kebijakan penempatan

Gunakan kebijakan penempatan untuk menentukan lokasi instance hardware Anda. Kebijakan penempatan tersebar memberikan keandalan yang lebih tinggi dengan menempatkan instance pada hardware yang berbeda, sehingga mengurangi dampak dari kegagalan hardware. Kebijakan penempatan rapat memberikan latensi yang lebih rendah di antara node dengan mendekatkan instance dalam infrastruktur jaringan yang sama.

FAQ

Luaskan semua

Apa itu Compute Engine? Apa kegunaannya?

Compute Engine adalah produk Infrastructure as a Service yang menawarkan virtual machine (VM) fleksibel dan dikelola sendiri yang dihosting di infrastruktur Google. Compute Engine mencakup VM berbasis Linux dan Windows yang berjalan di KVM, opsi penyimpanan lokal dan tahan lama, serta API berbasis REST sederhana untuk konfigurasi dan kontrol. Layanan ini terintegrasi dengan teknologi Google Cloud seperti Cloud Storage, App Engine, dan BigQuery untuk melakukan lebih dari sekadar kemampuan komputasi dasar untuk membuat aplikasi yang lebih kompleks dan canggih.

Apa yang dimaksud dengan CPU virtual di Compute Engine?

Di Compute Engine, setiap CPU virtual (atau vCPU) diimplementasikan sebagai hardware hyper-thread tunggal di salah satu Platform CPU yang tersedia. Pada prosesor Intel Xeon, Intel Hyper-Threading Technology memungkinkan beberapa thread aplikasi berjalan pada setiap core prosesor fisik. Anda mengonfigurasi VM Compute Engine dengan satu atau beberapa hyper-thread ini sebagai vCPU. Jenis mesin menentukan jumlah vCPU yang dimiliki instance Anda.

Bagaimana App Engine dan Compute Engine saling berhubungan?

Menurut kami, keduanya saling melengkapi. App Engine adalah penawaran Platform as a Service dari Google dan Compute Engine adalah penawaran Infrastructure as a Service dari Google. App Engine adalah platform yang bagus untuk menjalankan aplikasi berbasis web, aplikasi bisnis, dan backend seluler. Compute Engine sangat cocok digunakan saat Anda membutuhkan lebih banyak kontrol atas infrastruktur yang mendasarinya. Misalnya, Anda mungkin menggunakan Compute Engine saat memiliki logika bisnis yang mudah disesuaikan atau Anda ingin menjalankan sistem penyimpanan milik sendiri.
